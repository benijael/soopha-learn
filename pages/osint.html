// 01 · C'est quoi l'OSINT

Open Source Intelligence

🎯 DÉFINITION
Collecte et analyse d'informations provenant de sources publiques et accessibles librement.
  • Sites web, réseaux sociaux, forums
  • Bases de données publiques (WHOIS, registres)
  • Documents officiels, photos, vidéos
  • Métadonnées de fichiers
🛡️ QUI UTILISE L'OSINT
  • Pentesters — phase de reconnaissance avant une attaque
  • Journalistes d'investigation — Bellingcat, enquêtes
  • Services de renseignement — veille stratégique
  • RH & recruteurs — vérification de candidats
  • Attaquants — collecter des infos sur une cible
⚖️ Légal si on utilise des sources publiques et à des fins éthiques.
// 02 · Google Dorks

Google Dorks — L'art de chercher

🔍 OPÉRATEURS CLÉS
site:site:gabon.gov.ga — pages d'un domaine
filetype:filetype:pdf rapport confidentiel
intitle:intitle:"index of" /passwords
inurl:inurl:admin login
cache:cache:site.com — version archivée
"...":"mot de passe" site:.ga — phrase exacte
⚡ DORKS DANGEREUX
intitle:"index of" /backup
→ Répertoires de backup exposés

filetype:sql "password" site:.ga
→ Dumps SQL avec mots de passe

inurl:"/wp-content/uploads/" filetype:pdf
→ Docs confidentiels sur WordPress

"DB_PASSWORD" filetype:env
→ Fichiers .env avec credentials
⚠️ Ces infos sont publiques — mais les exploiter sans autorisation = illégal
// 03 · WHOIS & DNS

WHOIS & DNS — La carte d'identité des domaines

🌐 WHOIS
Base de données publique qui révèle qui possède un nom de domaine.
$ whois soopha-network.com
Registrant: SooPHA Group
Email: contact@soopha.com
Created: 2023-01-15
Registrar: IONOS SE
💡 Révèle propriétaire, email, hébergeur, dates
📡 DNS RECON
$ nslookup soopha-learn.com
A: 76.76.21.21 (Vercel)

$ dig soopha-learn.com MX
MX: mail.ionos.com (hébergeur email)

$ subfinder -d soopha-learn.com
api.soopha-learn.com
admin.soopha-learn.com ← intéressant !
💡 Révèle infrastructure, sous-domaines, emails
// 04 · Shodan

Shodan — Le moteur de recherche des hackers

🔴 C'EST QUOI SHODAN
Shodan indexe tout ce qui est connecté à Internet : serveurs, caméras, routeurs, IoT, systèmes industriels — avec leurs ports et services ouverts.
Shodan query : country:GA port:22
→ Serveurs SSH au Gabon

Shodan query : webcam city:"Libreville"
→ Caméras non sécurisées à Libreville

Shodan query : org:"SEEG" port:80
→ Serveurs web exposés d'un OIV !
📊 CE QUE SHODAN RÉVÈLE
  • Versions logicielles — identifier les systèmes non patchés
  • Services exposés — FTP anonyme, Telnet, bases de données sans auth
  • Certificats SSL — expirés ou auto-signés
  • Systèmes SCADA — infrastructures industrielles accessibles
UTILISATION ÉTHIQUE
  • Vérifier l'exposition de sa propre organisation
  • Recherche en cybersécurité et cartographie des menaces

Social Media Intelligence (SOCMINT)

👤 CE QU'ON PEUT TROUVER
  • LinkedIn — poste, entreprise, collègues, technologies utilisées
  • Facebook / Instagram — localisation, cercle social, routine
  • Twitter / X — opinions, affiliations, horaires d'activité

filename:.env "DB_PASSWORD"

  • SpiderFoot — OSINT automatisé multi-sources
  • theHarvester — emails, sous-domaines, IPs
$ sherlock benijael_mbata
Twitter: found
LinkedIn: found
GitHub: found
// 06 · Métadonnées

Métadonnées — Les données cachées dans vos fichiers

📁 C'EST QUOI
Des informations invisibles à l'œil nu stockées dans les fichiers : photos, documents, vidéos.
📸 PHOTO
GPS (lat/long), date, appareil, modèle, logiciel
📄 DOCUMENT
Auteur, entreprise, dates, révisions, nom PC
⚠️ EXEMPLES RÉELS
  • John McAfee localisé par une photo Instagram (GPS dans les EXIF)
  • Journaliste anonyme démasqué par les métadonnées Word
  • Localisation d'une base militaire via photos de soldats
PROTECTION
$ exiftool -all= photo.jpg
→ Supprime toutes les métadonnées
Signal supprime automatiquement les EXIF des photos envoyées.
// 07 · Wayback Machine & Archives

L'internet n'oublie jamais

🕰️ WAYBACK MACHINE
archive.org conserve des snapshots de milliards de pages web depuis les années 90. Même les pages supprimées restent accessibles.
  • Retrouver d'anciens contenus supprimés
  • Voir les anciennes versions d'un site
  • Trouver des emails et identifiants exposés par le passé
  • Analyser l'évolution d'une organisation
🗄️ AUTRES SOURCES D'ARCHIVES
  • Pastebin / GitHub — credentials leakés par erreur
  • HaveIBeenPwned — vérifier si un email est dans un leak
  • LinkedIn Sales Navigator — historique des postes
  • Google Cache — version cachée récente d'une page
  • Certificats SSL (crt.sh) — tous les sous-domaines émis
$ curl crt.sh/?q=%.soopha-learn.com
→ Tous les sous-domaines avec certificat
// 08 · OSINT Gabon & Afrique

OSINT — Contexte africain et gabonais

🇬🇦 SURFACE D'ATTAQUE AU GABON
  • Expositions fréquentes — portails admin accessibles publiquement
  • Beaucoup de sites .ga avec des configurations par défaut
  • Fichiers de sauvegarde (.sql, .bak) indexés par Google
  • Emails d'agents publics facilement trouvables — spear phishing facile
✅ COMMENT SE PROTÉGER
  • Faire un audit OSINT de sa propre organisation régulièrement
  • Configurer Google Alerts sur le nom de son entreprise
  • Vérifier ses données sur HaveIBeenPwned
  • Désactiver l'indexation des répertoires sur les serveurs web
  • Activer la protection WHOIS (privacy protection)
