C'est quoi un Réseau Informatique ?

La base absolue — comprendre ce qui se passe quand tu envoies un message

🌐 Définition
  • Un réseau = un ensemble d'appareils interconnectés qui échangent des données selon des règles communes appelées protocoles
  • Chaque appareil est un nœud : PC, smartphone, routeur, serveur, caméra IoT, imprimante...
  • Les données voyagent découpées en paquets — comme une lettre découpée en tranches, réassemblées à l'arrivée
  • Chaque nœud a une adresse unique — l'adresse IP — comme un numéro de rue dans une ville
  • Un réseau sans sécurité = une ville sans serrures ni gardiens
📦 Anatomie d'un paquet réseau
┌─────────────────────────────────┐
│ En-tête (Header) │
│ IP source : 192.168.1.10 │
│ IP destination : 8.8.8.8 │
│ Port source : 54321 │
│ Port destination : 443 (HTTPS) │
├─────────────────────────────────┤
│ Données (Payload) │
│ [contenu chiffré TLS...] │
└─────────────────────────────────┘
Un attaquant qui intercepte le paquet voit l'en-tête (métadonnées) — le payload est chiffré en HTTPS.
"Quand tu envoies un WhatsApp, ton message est découpé en paquets qui traversent des routeurs à travers plusieurs pays avant d'arriver chez ton contact. Chaque routeur est un point de passage — et un point de risque potentiel."
// Slide 03 · Protocoles · Les règles du réseau

Les Protocoles — Les Règles du Jeu

Sans protocoles, les machines ne pourraient pas se comprendre — comme parler sans langue commune

TCP — Transmission Control Protocol
Protocole orienté connexion — garantit la livraison dans l'ordre, avec accusé de réception. Si un paquet est perdu, il est retransmis. Plus lent mais fiable.
HTTP, HTTPS, SSH, SMTP, FTP → utilisent TCP
UDP — User Datagram Protocol
Protocole sans connexion — envoie et oublie, pas d'accusé de réception. Paquets peuvent arriver désordonnés ou se perdre. Plus rapide, tolérant aux pertes.
DNS, DHCP, VoIP, streaming vidéo, jeux → utilisent UDP
IP — Internet Protocol
Le protocole d'adressage et de routage — attribue des adresses uniques (IPv4/IPv6) et détermine le chemin que prennent les paquets à travers le réseau.
IPv4 : 192.168.1.1 · IPv6 : 2001:db8::1
ICMP — Internet Control Message Protocol
Protocole de diagnostic réseau — utilisé par ping et traceroute. Envoie des messages d'erreur et de contrôle entre équipements. Souvent bloqué par les firewalls.
ping 8.8.8.8 · traceroute google.com
ARP — Address Resolution Protocol
Résout les adresses IP en adresses MAC sur un réseau local. Chaque appareil maintient une table ARP. Vulnérable à l'ARP Spoofing — l'attaque MITM du réseau local.
arp -a → voir la table ARP · arping
DNS — Domain Name System
Traduit les noms de domaine en adresses IP — l'annuaire d'Internet. Sans DNS, il faut taper 142.250.185.14 au lieu de google.com. Vulnérable au DNS Spoofing / Cache Poisoning.
nslookup soopha-learn.com · dig google.com
// Slide 04 · Modèle OSI · 7 couches · Protocoles · Attaques

Le Modèle OSI — 7 Couches en Détail

Open Systems Interconnection — le modèle de référence qui structure toute communication réseau

7 Application Interface avec l'utilisateur — services réseau finaux HTTP · FTP · DNS · SMTP · SSH XSS · SQLi · phishing
6 Présentation Chiffrement, compression, formatage des données TLS/SSL · JPEG · MIME · ASCII SSL Strip · cert forgery
5 Session Ouvre, gère et ferme les sessions entre applications NetBIOS · RPC · SMB sessions Session hijacking
4 Transport Livraison fiable end-to-end, ports, contrôle de flux TCP · UDP · SCTP SYN Flood · port scan
3 Réseau Adressage logique IP et routage entre réseaux IP · ICMP · ARP · OSPF · BGP IP spoofing · routing attacks
2 Liaison Transfert fiable sur le lien physique, adresses MAC Ethernet · WiFi · PPP · VLAN ARP Spoofing · VLAN hopping
1 Physique Transmission des bits sur le support physique câble · fibre · WiFi · USB wiretapping · jamming

Le Modèle OSI en Pratique — À Quelle Couche Intervenir ?

Chaque problème réseau, chaque attaque, chaque outil opère à une couche précise

🛡️ Couche de chaque outil de sécurité
Firewall réseau → Couche 3 (IP) + Couche 4 (TCP/UDP)
Firewall applicatif (WAF) → Couche 7 (HTTP)
Wireshark / tcpdump → Couche 2 à Couche 7
Nmap (port scan) → Couche 3 + 4
TLS/HTTPS → Couche 6 (chiffrement)
VLAN → Couche 2 (Liaison)
VPN (tunnel) → Couche 3 à 7
⚔️ Attaque → couche ciblée
ARP Spoofing → Couche 2
IP Spoofing → Couche 3
SYN Flood (DDoS) → Couche 4
SSL Strip → Couche 6
SQLi / XSS → Couche 7
Phishing → Couche 7
Wiretapping / écoute → Couche 1
"En cybersécurité, parler de 'couche' n'est pas du jargon — c'est une façon précise d'identifier où agir. Un firewall couche 3 ne protège pas contre une SQLi couche 7. Chaque menace a sa couche, chaque défense a la sienne."

OSI = modèle théorique de référence · TCP/IP = ce qu'Internet utilise vraiment

🔄 Encapsulation — comment un paquet se construit
  • Couche Application génère les données (ex: requête HTTP GET /index.html)
  • Couche Transport ajoute un en-tête TCP (ports source/dest, numéro de séquence)
  • Couche Internet ajoute un en-tête IP (adresses IP source et destination)
  • Couche Accès ajoute un en-tête Ethernet (adresses MAC + CRC de vérification)
  • À la réception, chaque couche retire son en-tête et passe les données à la couche supérieure — c'est la déencapsulation
// Slide 07 · TCP vs UDP · Three-way handshake · QCM requis

TCP vs UDP — Comprendre la Différence en Profondeur

Le choix entre TCP et UDP détermine comment les données voyagent — et comment les attaquer

TCP — Fiable mais lent
  • Three-way handshake obligatoire avant tout échange : SYN → SYN-ACK → ACK
  • Accusés de réception (ACK) — chaque paquet reçu est confirmé
  • Retransmission automatique des paquets perdus
  • Ordre garanti — les paquets sont réordonnés si nécessaire
  • Vulnérable au SYN Flood — inonder avec des SYN sans ACK final
UDP — Rapide mais sans garantie
  • Pas de handshake — on envoie directement sans établir de connexion
  • Pas d'accusé de réception — fire and forget
  • Paquets peuvent se perdre ou arriver dans le désordre
  • Latence minimale — idéal pour la VoIP, les jeux, le streaming
  • Vulnérable au UDP Flood et à l'amplification DNS
# Le Three-way handshake TCP — établir une connexion
Client → Serveur : SYN         ; "Je veux me connecter" (seq=x)
Serveur → Client : SYN-ACK   ; "OK, je suis prêt" (seq=y, ack=x+1)
Client → Serveur : ACK         ; "Connexion établie" (ack=y+1)
# → SYN Flood : envoyer des milliers de SYN sans jamais envoyer l'ACK final
# → Le serveur garde toutes ces demi-connexions en mémoire → saturation
// Slide 08 · LAN · WAN · MAN · PAN · WLAN · Types de réseaux

Types de Réseaux — Du Bluetooth à Internet

Chaque type de réseau a une portée, une technologie et un profil de risque différents

LAN — Local Area Network
Réseau local — bâtiment, bureau, maison. Portée : quelques centaines de mètres. Technologie : Ethernet (câble) ou WiFi (sans fil). Haut débit, faible latence.
Réseau de bureau · réseau domestique · salle informatique
WAN — Wide Area Network
Réseau étendu — relie des LAN à travers des villes, pays ou continents. Internet est le plus grand WAN. Géré par des opérateurs télécoms. Latence plus élevée.
Internet · liaisons MPLS entreprise · connexions transatlantiques
MAN — Metropolitan Area Network
Réseau métropolitain — couvre une ville ou une agglomération. Entre LAN et WAN. Utilisé par les opérateurs pour relier les quartiers, les campus universitaires, les mairies.
Réseau fibré d'une ville · campus universitaire multi-sites
PAN — Personal Area Network
Réseau personnel — autour d'une personne, portée de quelques mètres. Technologies : Bluetooth, NFC, infrarouge. Risque BlueSnarfing (vol de données Bluetooth).
Smartphone + casque Bluetooth · paiement NFC · montre connectée
WLAN — Wireless LAN
LAN sans fil (WiFi). Standard IEEE 802.11. Versions : WiFi 4 (802.11n), WiFi 5 (ac), WiFi 6 (ax). Chiffrement : WEP (cassé), WPA2, WPA3. Surface d'attaque élevée.
WiFi domicile · hotspot café · réseau d'entreprise sans fil
VPN — Virtual Private Network
Tunnel chiffré qui crée un réseau privé sur Internet. L'employé en télétravail rejoint le LAN de l'entreprise via VPN. Protocoles : OpenVPN, WireGuard, IPsec.
Télétravail sécurisé · contournement géo-restrictions · anonymat
// Slide 09 · MAC vs IP · Adressage · Couche 2 vs Couche 3

MAC vs IP — Deux Niveaux d'Adressage

L'adresse MAC identifie le matériel · l'adresse IP identifie l'emplacement réseau

🔌 Adresse MAC — Couche 2
  • Media Access Control — gravée dans la carte réseau à la fabrication
  • Format : 6 octets en hexadécimal — ex : AA:BB:CC:11:22:33
  • Les 3 premiers octets = OUI — identifiant du fabricant (Cisco, Apple, Samsung...)
  • Utilisée uniquement sur le réseau local (LAN) — invisible sur Internet
  • MAC Spoofing — possible de falsifier son adresse MAC pour contourner les contrôles d'accès WiFi
🌐 Adresse IP — Couche 3
  • Internet Protocol — attribuée dynamiquement (DHCP) ou statiquement
  • IPv4 : 4 octets en décimal — ex : 192.168.1.10 (4 milliards d'adresses)
  • IPv6 : 16 octets en hex — ex : 2001:0db8::1 (340 undécillions)
  • Routée sur Internet et les réseaux locaux
  • IP Spoofing — falsifier l'IP source pour masquer son origine ou usurper une identité
$ ip addr show eth0
2: eth0: <BROADCAST,MULTICAST,UP>
  link/ether aa:bb:cc:11:22:33 brd ff:ff:ff:ff:ff:ff ← MAC
  inet 192.168.1.10/24 brd 192.168.1.255          ← IPv4
  inet6 fe80::a8bb:ccff:fe11:2233/64              ← IPv6 link-local
// Slide 10 · Topologies réseau · QCM requis

Topologies Réseau — Comment les Machines Sont Connectées

La topologie détermine la résilience, les performances et la surface d'attaque

🚌
Bus
Toutes les machines partagent un seul câble commun. Un paquet envoyé = toutes les machines le voient. Obsolète mais important historiquement.
⚠ Panne du câble = tout le réseau tombe · sniffing facile
⭐
Étoile
Toutes les machines connectées à un switch ou hub central. La plus courante dans les bureaux et maisons. Facile à dépanner et à étendre.
⚠ Switch central = point de défaillance unique et cible idéale
💠
Maillé (Mesh)
Chaque nœud connecté à plusieurs autres. Très résilient — si un lien tombe, les données empruntent un autre chemin. Utilisé par Internet lui-même.
✅ Haute résilience · utilisé pour les datacenters critiques
🔗
Anneau (Ring)
Les machines forment un anneau, les données circulent dans un sens. Une rupture interrompt tout le réseau. Utilisé en fibre optique (SONET/SDH).
⚠ Rupture du lien = réseau coupé · difficile à déboguer
🌳
Arbre (Tree)
Hiérarchique — un switch racine alimente des switchs secondaires qui alimentent des machines. Très utilisé dans les grandes entreprises et les campus.
// Facile à scaler · switch racine = point critique
🔀
Hybride
Combinaison de plusieurs topologies. La plupart des réseaux réels sont hybrides : étoile au niveau local, maillé au niveau WAN/datacenter.
✅ La réalité des grands réseaux d'entreprise
// Slide 11 · Équipements réseau · Hub · Switch · Routeur · Firewall

Les Équipements Réseau — Qui Fait Quoi ?

Chaque équipement opère à une couche OSI précise — et a ses propres vulnérabilités

Hub — Couche 1
Répéteur multiport — reçoit un paquet et le retransmet sur TOUS les ports sans distinction. Chaque machine voit le trafic de toutes les autres. Totalement obsolète mais encore présent dans les vieilles infras africaines.
⚠ Sniffing passif trivial — Wireshark voit tout
Switch — Couche 2
Commutateur intelligent — apprend les adresses MAC et envoie les paquets uniquement au bon port. Remplace le hub dans tous les réseaux modernes. Peut être segmenté en VLANs.
⚠ Vulnérable au MAC flooding, ARP spoofing, VLAN hopping
Routeur — Couche 3
Achemine les paquets entre différents réseaux en se basant sur les adresses IP. C'est lui qui connecte ton réseau local à Internet. Dispose d'une table de routage maintenue dynamiquement.
⚠ Attaques BGP hijacking, faille dans le firmware
Firewall — Couches 3-7
Filtre le trafic selon des règles définies. Stateless (filtre paquet par paquet), Stateful (suit l'état des connexions), NGFW (inspection applicative, DPI, IPS intégré).
✅ Première ligne de défense — bien configuré, c'est essentiel
Point d'accès WiFi — Couches 1-2
Crée un réseau sans fil (WLAN) et le relie au réseau câblé. Le chiffrement WPA3 est le standard actuel. WEP est totalement cassé, WPA2 vulnérable à PMKID et KRACK.
⚠ WEP crackable en minutes · réseau ouvert = catastrophe
Proxy — Couche 7
Intermédiaire entre les clients et Internet. Forward proxy = cache + filtrage URL. Reverse proxy = load balancer + protection serveur. WAF = proxy spécialisé sécurité web.
Nginx, HAProxy, Squid, Cloudflare (reverse proxy)
// Slide 12 · Routage · Traceroute · Voyage d'un paquet

Le Voyage d'un Paquet — De Kinshasa à Google

Tracer le chemin d'un paquet pour comprendre le réseau — et identifier les points de surveillance

1
Ton navigateur crée une requête HTTP GET / — la couche Transport ajoute l'en-tête TCP (port 443), la couche IP ajoute ton IP source et l'IP Google destination
2
Ton routeur domicile reçoit le paquet, vérifie sa table de routage → "l'IP 142.250.x.x n'est pas dans mon réseau local → je l'envoie vers mon opérateur" (NAT appliqué)
3
L'opérateur télécoms reçoit le paquet, le route vers un point d'échange Internet (IXP) régional — en Afrique : IXPN Nigeria, JINX Johannesburg, DE-CIX Djibouti
4
Les câbles sous-marins transportent le paquet vers l'Europe ou les USA si aucun datacenter Google n'est plus proche — chaque saut = un routeur traversé
5
Google reçoit, traite et répond — la réponse suit le chemin inverse. Traceroute montre chaque routeur traversé avec sa latence
$ traceroute google.com
1  192.168.1.1    (routeur local)     1ms
2  10.x.x.x      (opérateur)        8ms
3  196.x.x.x    (IXP Afrique)     45ms
4  * * *         (routeur silencieux)  ---
5  142.250.x.x  (Google)           120ms
