Une adresse IP = 32 bits organisés en 4 octets — comprendre le binaire, c'est tout comprendre

🔢 Convertir binaire ↔ décimal
# Table des puissances de 2 (1 octet)
128 64 32 16 8 4 2 1
1 1 0 0 0 0 0 0 = 128+64 = 192
1 0 1 0 1 0 0 0 = 128+32+8 = 168
0 0 0 0 1 0 1 0 = 8+2 = 10

# Astuce : un octet = 0 à 255 maximum
# 11111111 binaire = 255 décimal
📏 Caractéristiques IPv4
  • 32 bits = 4 octets = ~4,3 milliards d'adresses théoriques
  • Format pointé décimal — 4 nombres entre 0 et 255
  • Deux parties : partie réseau (Network ID) + partie hôte (Host ID)
  • Le masque de sous-réseau définit où s'arrête la partie réseau
  • IPv4 est épuisé depuis 2011 — d'où la migration vers IPv6

// Slide 04 · Masque de sous-réseau · Notation CIDR · /24 /16 /8

Masque de Sous-réseau & Notation CIDR

Le masque dit au routeur : "jusqu'où va mon réseau" — le /24 est ton meilleur ami

🎭 Comment fonctionne le masque
  • Le masque est un nombre de 32 bits avec des 1 pour la partie réseau et des 0 pour la partie hôte
  • /24 = 24 bits à 1 = 255.255.255.0 en décimal
  • Pour 192.168.1.10/24 : le réseau = 192.168.1, l'hôte = .10
  • Opération AND entre IP et masque → donne l'adresse réseau
  • CIDR = Classless Inter-Domain Routing — notation compacte standard depuis 1993
📐 Masques les plus courants
/8 = 255.0.0.0     → 16 777 214 hôtes
/16 = 255.255.0.0   → 65 534 hôtes
/24 = 255.255.255.0 → 254 hôtes ← le plus courant
/25 = 255.255.255.128→ 126 hôtes
/26 = 255.255.255.192→ 62 hôtes
/27 = 255.255.255.224→ 30 hôtes
/28 = 255.255.255.240→ 14 hôtes
/30 = 255.255.255.252→ 2 hôtes (liaison P2P)
/32 = 255.255.255.255→ 1 hôte (route spécifique)
"Formule magique : nombre d'hôtes = 2^(32-préfixe) - 2. Pour un /24 : 2^8 - 2 = 254 hôtes. On enlève 2 car l'adresse réseau (.0) et l'adresse broadcast (.255) ne sont pas assignables à des machines."
// Slide 05 · Subnetting · Calcul · Exercices · QCM requis

Subnetting — Découper un Réseau

La compétence qui différencie un technicien réseau d'un administrateur — calculer sans calculatrice

🔪 Exercice concret : découper 192.168.10.0/24 en 4 sous-réseaux égaux
1
Déterminer le nouveau préfixe : 4 sous-réseaux = 2² → emprunter 2 bits supplémentaires → /24 + 2 = /26
2
Calculer les hôtes par sous-réseau : 2^(32-26) - 2 = 2^6 - 2 = 62 hôtes utilisables par sous-réseau
3
Calculer l'incrément : 256 - 192 (valeur du masque /26 dans le dernier octet) = 64 → chaque sous-réseau fait un bloc de 64 adresses
4
Lister les 4 sous-réseaux :
192.168.10.0/26 → hôtes .1 à .62, broadcast .63
192.168.10.64/26 → hôtes .65 à .126, broadcast .127
192.168.10.128/26 → hôtes .129 à .190, broadcast .191
192.168.10.192/26 → hôtes .193 à .254, broadcast .255
// Slide 06 · Broadcast · Loopback · APIPA · Adresses réservées

Adresses Spéciales — Celles qui ont un Rôle Particulier

Pas toutes les IPs sont pour les machines — certaines ont des fonctions critiques

Broadcast — 255.255.255.255
Envoie un message à toutes les machines du réseau local simultanément. Le routeur ne le transmet jamais au-delà du LAN. Utilisé par DHCP Discovery, ARP Request.
192.168.1.255 = broadcast du réseau 192.168.1.0/24
Loopback — 127.0.0.1
S'adresse à ta propre machine — le paquet ne sort jamais de la carte réseau. Tester si la pile TCP/IP fonctionne. 127.0.0.0/8 entier est réservé loopback.
ping 127.0.0.1 · curl http://localhost:3000
APIPA — 169.254.0.0/16
Automatic Private IP Addressing — attribuée automatiquement quand le DHCP est inaccessible. Si tu vois une IP 169.254.x.x, ton appareil n'a pas pu obtenir d'IP. Signe d'un problème réseau.
169.254.x.x → DHCP unreachable → pas d'Internet
Multicast — 224.0.0.0/4
Envoyer à un groupe spécifique de machines qui se sont abonnées. Utilisé pour le streaming vidéo, les protocoles de routage (OSPF : 224.0.0.5, RIP : 224.0.0.9).
224.0.0.1 = tous les hôtes · 224.0.0.2 = tous les routeurs
Adresse réseau — .0
La première adresse d'un bloc CIDR identifie le réseau lui-même — non assignable à un hôte. Ex : 192.168.1.0 est l'identifiant du réseau 192.168.1.0/24.
192.168.1.0 → identifie le réseau, jamais une machine
0.0.0.0
Route par défaut (default gateway) — "envoyer tout ce qui ne correspond à aucune route connue vers cette adresse". Aussi utilisée par DHCP pour signifier "n'importe quelle IP source".
0.0.0.0/0 = default route → passe tout au routeur
// Slide 07 · NAT · IP privée vs IP publique · QCM requis

NAT — Network Address Translation

Pourquoi ton adresse locale (192.168.x.x) et ton IP publique sont différentes — et comment ça marche

🔄 Comment fonctionne le NAT
  • Ton routeur possède une seule IP publique attribuée par ton FAI (ex: 41.243.x.x en Afrique)
  • Toutes tes machines ont des IPs privées (192.168.1.x) non routables sur Internet
  • Quand tu accèdes à Google, le routeur remplace ton IP privée par son IP publique + note le port dans sa table NAT
  • La réponse de Google arrive sur l'IP publique → le routeur consulte sa table → redirige vers ta machine
  • NAT permet à 254 machines de partager 1 seule IP publique — économie d'adresses IPv4
🔢 Types de NAT
# SNAT / Masquerading (le plus courant)
Plusieurs IPs privées → 1 IP publique
192.168.1.10:54321 → 41.243.x.x:54321
192.168.1.11:60000 → 41.243.x.x:60000

# DNAT / Port Forwarding
IP publique:port → IP privée:port
41.243.x.x:80 → 192.168.1.100:80
# Exposer un serveur web interne

# Voir son IP publique
$ curl ifconfig.me
"Le NAT a sauvé Internet — en 2011, l'IANA a attribué le dernier bloc IPv4 disponible. Sans NAT, chaque appareil aurait besoin d'une IP publique unique et nous aurions manqué d'adresses bien avant. IPv6 est la vraie solution à long terme."
// Slide 08 · DHCP · Attribution automatique · DORA

DHCP — L'Attribution Automatique des IPs

Dynamic Host Configuration Protocol — comment ton téléphone obtient une IP en se connectant au WiFi

📋 Ce que DHCP attribue
  • Adresse IP — une IP libre de la plage configurée (ex: 192.168.1.50)
  • Masque de sous-réseau — ex: 255.255.255.0 (/24)
  • Passerelle par défaut — l'IP du routeur (ex: 192.168.1.1)
  • Serveurs DNS — ex: 8.8.8.8 (Google) ou 1.1.1.1 (Cloudflare)
  • Durée du bail (lease) — combien de temps l'IP est réservée (24h, 7 jours...)
  • Un serveur DHCP rogue peut attribuer de fausses infos → attaque DHCP Starvation
🤝 Le processus DORA
DISCOVER — broadcast UDP port 67
Client → 255.255.255.255 : "Y a-t-il un serveur DHCP ?"

OFFER — serveur répond
DHCP → Client : "Je t'offre 192.168.1.50, bail 24h"

REQUEST — client accepte
Client → broadcast : "J'accepte 192.168.1.50"

ACK — confirmation
DHCP → Client : "Confirmé, IP réservée 24h"

$ ip addr · ipconfig /all ← voir l'IP obtenue

IPv6 — L'Avenir du Réseau

128 bits · 340 undécillions d'adresses · Déjà déployé en Afrique — comprendre l'essentiel

📐 Format et structure IPv6
  • 128 bits = 8 groupes de 4 chiffres hexadécimaux séparés par :
  • Exemple : 2001:0db8:85a3:0000:0000:8a2e:0370:7334
  • Simplification : les zéros consécutifs → :: (une seule fois par adresse)
  • Exemple simplifié : 2001:db8:85a3::8a2e:370:7334
  • Préfixe réseau le plus courant : /64 (moitié réseau, moitié interface)
  • 340 000 000 000 000 000 000 000 000 000 000 000 000 adresses — jamais à court
🔑 Adresses IPv6 importantes
::1                  loopback (= 127.0.0.1)
fe80::/10           link-local (auto-configuré)
2001:db8::/32      documentation/exemples
2000::/3            adresses globales (Internet)
fc00::/7            unique local (= RFC 1918)
ff02::1            multicast tous hôtes LAN

# Voir ses adresses IPv6
$ ip -6 addr show
$ ping6 ::1
Afrique : ~12% de trafic IPv6 (2024, en croissance) Dual-stack = IPv4 + IPv6 en parallèle
// Slide 11 · IPv6 vs IPv4 · Nouveautés · Sécurité

IPv6 vs IPv4 — Ce qui Change Vraiment

IPv6 n'est pas juste "plus d'adresses" — c'est une refonte complète du protocole

Adressage illimité
340 undécillions d'adresses — chaque grain de sable sur Terre pourrait avoir des milliards d'adresses. Plus besoin de NAT — chaque device a une vraie IP publique unique.
Fin du NAT → chaque IoT device directement adressable sur Internet
Auto-configuration (SLAAC)
Stateless Address Autoconfiguration — un device peut générer sa propre adresse IPv6 depuis son adresse MAC et le préfixe réseau, sans DHCP. Plug and play réseau.
Préfixe réseau + EUI-64 (dérivé MAC) → IP auto-générée
IPsec natif
IPv6 intègre IPsec (chiffrement et authentification) nativement dans la spécification. En IPv4, IPsec est optionnel. Les communications IPv6 peuvent être chiffrées end-to-end par défaut.
Authentification et chiffrement sans VPN supplémentaire
En-tête simplifié
L'en-tête IPv6 est plus simple qu'IPv4 malgré des adresses plus longues — la checksum a été supprimée (délégué aux couches supérieures). Les routeurs traitent les paquets plus rapidement.
En-tête fixe de 40 octets vs variable en IPv4
Multicast et Anycast
IPv6 remplace le broadcast par le multicast (ciblé) et ajoute l'anycast (envoi au plus proche d'un groupe). Plus efficace, moins de bruit sur le réseau.
Anycast : DNS Cloudflare 2606:4700:4700::1111
Nouveaux risques de sécurité
Pas de NAT = les devices sont directement exposés. NDP (remplace ARP) est vulnérable au NDP Spoofing. Beaucoup d'équipements africains mal configurés pour IPv6.
⚠ Activer IPv6 sans firewall = surface d'attaque maximale

Diagnostic Réseau — Identifier les Problèmes d'Adressage

Les commandes que tout technicien réseau utilise quotidiennement

🔧 Commandes essentielles Linux
# Voir sa configuration IP
$ ip addr show
$ ip route show     # table de routage

# Tester la connectivité
$ ping 8.8.8.8      # ICMP vers Google DNS
$ ping6 ::1         # test IPv6 loopback

# Tracer le chemin
$ traceroute google.com
$ mtr google.com    # ping + traceroute live

# Résolution DNS
$ nslookup soopha-learn.com
$ dig soopha-learn.com A

# IP publique
$ curl ifconfig.me

4

traceroute → où ça bloque dans le chemin ?

L'Adressage IP en Afrique — Les Réalités du Terrain

AFRINIC · pénurie IPv4 · adoption IPv6 · architecture typique des FAI africains

🌍 AFRINIC — Le registre africain
  • AFRINIC (African Network Information Centre) — distribue les adresses IP en Afrique depuis 2005, basé à Maurice
  • L'Afrique est la région RIR avec le moins d'IPv4 par habitant — historiquement désavantagée
  • Situation de crise : AFRINIC est en phase de pénurie sévère depuis 2021 — pool épuisé
  • IPv6 : plusieurs opérateurs africains leaders mondiaux en adoption (MTN, Orange Africa, Safaricom)
  • Gabon, Congo, Cameroun : adoption IPv6 encore lente — opportunité professionnelle
🏗️ Architecture typique FAI africain
Internet
  ↓ Câble sous-marin (ACE, WACS, SAT3)
FAI (IXP national)
  ↓ IP publique attribuée par AFRINIC
CGNAT (Carrier-Grade NAT)
  ← Des milliers de clients derrière 1 IP pub
  ↓ IP privée 10.x.x.x attribuée au client
Routeur client (box FAI)
  ↓ DHCP local
192.168.1.x ← tes appareils
