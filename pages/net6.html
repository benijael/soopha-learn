Attaques Réseau — Comprendre pour Défendre

Les attaques les plus courantes sur les réseaux locaux et Internet — mécanismes et contre-mesures

ARP Spoofing → MITM
Envoyer de faux ARP Reply pour associer sa MAC à l'IP de la passerelle. Tout le trafic de la victime passe par l'attaquant. Outil : arpspoof, Ettercap, Bettercap.
✅ DAI (Dynamic ARP Inspection) sur switches · entrées ARP statiques · VPN
DNS Cache Poisoning
Injecter de fausses réponses DNS dans le cache d'un résolveur pour rediriger les utilisateurs vers un faux site. Attaque Kaminsky exploitait le port source prévisible.
✅ DNSSEC · randomisation port source · DNS over HTTPS
SYN Flood (DDoS couche 4)
Envoyer des millions de SYN avec des IPs spoofées. La table des half-open connections du serveur sature. Aucun ACK final → le serveur attend pendant le timeout.
✅ SYN Cookies · rate limiting TCP · scrubbing center · Cloudflare
SSL Strip
Dans un MITM, l'attaquant remplace les liens HTTPS par HTTP entre la victime et le site. La victime pense être en HTTP normal, l'attaquant voit tout en clair.
✅ HSTS (HTTP Strict Transport Security) · HSTS Preloading
VLAN Hopping
Exploiter la mauvaise configuration d'un switch pour accéder à des VLANs interdits. Technique du double-tagging 802.1Q ou switch spoofing (DTP). Couche 2.
✅ Désactiver DTP · changer Native VLAN · access ports explicites
DHCP Starvation + Rogue DHCP
1) Saturer le pool DHCP avec de fausses requêtes. 2) Installer un faux serveur DHCP qui distribue de fausses infos (fausse passerelle, faux DNS). MITM DHCP.
✅ DHCP Snooping sur switches · port security · 802.1X

Nmap Avancé — Cartographier et Analyser

Network Mapper — l'outil de reconnaissance réseau de référence, utilisé légalement en pentest et audit

🔍 Types de scans essentiels
-sS
SYN Scan (Stealth)
Envoie SYN, attend SYN-ACK (ouvert) ou RST (fermé). Ne complète pas le handshake → moins de logs
-sV
Version Detection
Détecte les versions des services : Apache 2.4.51, OpenSSH 8.9, MySQL 8.0...
-O
OS Detection
Fingerprinting TCP/IP pour identifier le système d'exploitation cible
-sU
UDP Scan
Scanner les ports UDP (DNS:53, SNMP:161, DHCP:67...) — plus lent
-A
Aggressive
-sV + -O + scripts NSE + traceroute — tout en une commande
-p-
All Ports
Scanner les 65535 ports (pas seulement les 1000 défaut)
🧩 Nmap Scripting Engine (NSE)
# Vulnérabilités connues
$ nmap --script vuln 192.168.1.0/24

# Détection SMB (EternalBlue)
$ nmap --script smb-vuln* -p 445 target

# Brute force SSH
$ nmap --script ssh-brute -p 22 target

# Bannières des services
$ nmap --script banner -p 21,22,25,80 target

# HTTP — titres des pages
$ nmap --script http-title -p 80,443 target

# Sauvegarde des résultats
$ nmap -A -oA scan_complet 192.168.1.0/24
# → .xml .nmap .gnmap générés
// Slide 04 · Evasion Firewall · Légalité · Timing · Décoys

Nmap — Évasion de Firewalls & Cadre Légal

Comment les firewalls bloquent Nmap — et comment contourner légalement dans un périmètre autorisé

🔀 Techniques d'évasion (pentest autorisé uniquement)
# Timing lent (T0=paranoïaque, T5=agressif)
$ nmap -T1 target  ← très lent, discret
$ nmap -T4 target  ← rapide (défaut = T3)

# Fragmenter les paquets
$ nmap -f target

# Décoys — mélanger sa vraie IP
$ nmap -D 1.2.3.4,5.6.7.8,ME target

# Changer le port source
$ nmap --source-port 53 target

# Scan depuis un zombie (Idle Scan)
$ nmap -sI zombie_ip target
⚖️ Cadre légal — TOUJOURS vérifier
  • Scanner son propre réseau — toujours légal et recommandé régulièrement
  • Bug bounty / pentest contractuel — légal dans le périmètre défini par écrit
  • Lab et CTF — toujours légal sur les machines qui t'appartiennent
  • Scanner un réseau sans autorisation écrite — illégal dans tous les pays. CFAA aux USA, Loi Godfrain en France, cybercriminalité en Afrique (UEMOA, UA)
  • Un scan Nmap laisse des traces dans les logs — toujours avoir un contrat signé
  • Les IPS modernes détectent et bloquent les scans Nmap — même lents

IDS & IPS — Détecter et Bloquer les Intrusions

Intrusion Detection vs Prevention System — comment ça marche et comment les configurer

🔍 IDS vs IPS — La différence fondamentale
  • IDS (Intrusion Detection System) — observe le trafic en copie (mode passif, TAP ou SPAN port), détecte les signatures malveillantes, génère des alertes. Ne bloque pas.
  • IPS (Intrusion Prevention System) — placé inline dans le flux réseau, détecte ET bloque en temps réel. Si l'IPS tombe en panne → le réseau peut tomber (fail-open vs fail-close).
  • NIDS — Network-based (surveille le réseau), HIDS — Host-based (surveille un système)
  • Suricata et Snort peuvent fonctionner en mode IDS ou IPS selon la configuration
📋 Règle Suricata — anatomie
# Structure d'une règle Suricata/Snort
alert  ← action (alert/drop/reject/pass)
tcp   ← protocole
any any ← src IP + port
-> $HOME_NET 22 ← dst (réseau interne port 22)
(msg:"SSH brute force";
 flags:S; ← SYN uniquement
 threshold: type threshold, track by_src,
  count 5, seconds 60;
 sid:1000001; rev:1;)

# Lancer Suricata en mode IDS
$ suricata -c /etc/suricata/suricata.yaml -i eth0
"Un IDS/IPS mal tuné est inutile — trop de faux positifs et les analystes l'ignorent, trop de faux négatifs et les attaques passent. Le tuning est 80% du travail : ajuster les seuils, exclure les IPs légitimes, adapter au contexte réseau."
// Slide 06 · fail2ban · Protection active · Rate limiting

Protection Active — Bloquer les Attaquants Automatiquement

fail2ban, rate limiting, CrowdSec — réagir en temps réel sans IPS complet

🚫 fail2ban — bannir les brute forceurs
# Installation
$ sudo apt install fail2ban

# Config /etc/fail2ban/jail.local
[sshd]
enabled = true
port = 22
maxretry = 3     ← 3 essais max
bantime = 3600  ← ban 1 heure
findtime = 600  ← fenêtre 10 min
ignoreip = 127.0.0.1 10.0.0.0/8

# Voir les IPs bannies
$ fail2ban-client status sshd

# Débannir une IP
$ fail2ban-client set sshd unbanip 1.2.3.4
⚡ CrowdSec — intelligence collective
  • CrowdSec = fail2ban moderne + communauté. Les IPs malveillantes détectées par des milliers de serveurs sont partagées — tu bénéficies de la threat intelligence collective
  • Analyse les logs en temps réel (SSH, Nginx, Apache, Traefik...) et ban automatiquement
  • Blocklists communautaires — les IPs connues des scanners, botnets, tor exit nodes
  • Installation : curl -s https://packagecloud.io/install/repositories/crowdsec/crowdsec/script.deb.sh | sudo bash && sudo apt install crowdsec
RATE LIMITING
# Nginx : 10 req/s max par IP
limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;
limit_req zone=api burst=20 nodelay;

VPN en Profondeur — WireGuard vs OpenVPN

Comprendre les VPN au niveau du protocole — choisir et configurer le bon pour chaque usage

⚡ WireGuard — Le VPN moderne
# Installer WireGuard
$ sudo apt install wireguard

# Générer les clés
$ wg genkey | tee privatekey | wg pubkey > publickey

# Config serveur /etc/wireguard/wg0.conf
[Interface]
Address = 10.0.0.1/24
ListenPort = 51820
PrivateKey = <clé privée serveur>
PostUp = iptables -A FORWARD -i wg0 -j ACCEPT; iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE

[Peer]
PublicKey = <clé publique client>
AllowedIPs = 10.0.0.2/32

# Démarrer
$ wg-quick up wg0

Hardening Réseau — Sécuriser de A à Z

La checklist complète pour durcir une infrastructure réseau — switch, routeur, WiFi, firewall

🔒 Switch managé
VLANs configurés — segmenter servers, users, IoT, guest WiFi
Port security — limiter les MACs par port (max 2-3)
DHCP Snooping activé — bloquer les faux serveurs DHCP
Dynamic ARP Inspection (DAI) — contre l'ARP Spoofing
Désactiver les ports inutilisés (shutdown)
STP BPDU Guard sur les ports access
Changer le Native VLAN de 1 vers un VLAN inutilisé
802.1X — authentification avant accès réseau
🔒 Routeur & Firewall
Firmware à jour — vérifier et patcher régulièrement
Changer les credentials par défaut (admin/admin → interdit)
Désactiver l'administration via WAN — management local ou VPN uniquement
Politique firewall Deny-All par défaut
Bloquer les bogons (adresses privées depuis Internet)
Logging activé — envoyer vers un SIEM ou syslog central
Désactiver UPnP et WPS (vecteurs d'attaque courants)
NTP synchronisé — timestamps corrects dans les logs
// Slide 09 · WiFi sécurisé · WPA3 · 802.1X · SSID · Rogue AP

Hardening WiFi — Le Sans-fil Sécurisé

Le WiFi est la surface d'attaque la plus exposée en entreprise africaine — le durcir correctement

Chiffrement WPA3 (ou WPA2+AES minimum)
WPA3 = résistant aux attaques dictionnaire offline (SAE au lieu de PSK). WPA2 avec AES-CCMP (pas TKIP) = acceptable. WEP = cassable en minutes.
WPA3-Personal ou WPA2-Enterprise (802.1X) en entreprise
802.1X — Authentification RADIUS
Chaque employé s'authentifie avec ses credentials AD/LDAP avant d'accéder au WiFi. Pas de mot de passe partagé — révocation immédiate quand un employé part.
FreeRADIUS + Active Directory · Cisco ISE · Aruba ClearPass
Segmentation WiFi invité
SSID invité sur un VLAN séparé (ex: VLAN 50) avec accès Internet uniquement. Client isolation activée — les invités ne se voient pas entre eux. Pas d'accès au LAN interne.
Portail captif · limitation de bande passante · expiration auto
Détection de Rogue AP
Un attaquant ou un employé branche un point d'accès non autorisé sur le réseau. Les systèmes WIDS (Wireless IDS) détectent les SSID non autorisés et les Evil Twin.
Kismet · Ekahau · Cisco WLC avec Rogue Detection
SSID et diffusion
Masquer le SSID (hidden) est inutile — détectable en quelques secondes avec airodump-ng. Utiliser un nom de SSID neutre sans identifier l'entreprise (pas "SOOPHA-Corp-HQ").
⚠ Hidden SSID = fausse sécurité · security through obscurity = fausse sécurité
PMF — Protected Management Frames
Protège les trames de management WiFi (deauth, disassoc) contre la déauthentification forcée. Sans PMF, un attaquant peut forcer la déconnexion de tous les clients (DoS WiFi).
Activer PMF/802.11w sur tous les points d'accès modernes
// Slide 10 · VPN usage · Split tunneling · DNS leaks · QCM requis

VPN — Ce qu'il Protège et Ce qu'il Ne Protège Pas

Beaucoup d'idées fausses circulent sur les VPN — comprendre exactement ce qu'un VPN fait

✅ Ce qu'un VPN protège réellement
  • Le trafic entre toi et le serveur VPN — chiffré, lisible par personne sur le réseau local
  • Ton adresse IP réelle — le site destination voit l'IP du serveur VPN
  • Tes requêtes DNS — si le VPN inclut un resolver DNS (sinon DNS leak)
  • Le trafic sur WiFi public — personne sur le réseau ne peut sniffertes données
  • Le VPN protège contre les intermédiaires réseau (FAI, café WiFi, MITM sur LAN)
🔴 Ce qu'un VPN ne protège PAS
  • Ton comportement en ligne — le serveur VPN voit tout ton trafic déchiffré
  • Les malwares déjà sur ton device — le VPN ne scanne pas les fichiers
  • Le phishing — une fausse page reste fausse avec ou sans VPN
  • La traçabilité si tu es connecté à tes comptes (Google, Facebook te traquent)
  • DNS leaks — si les requêtes DNS sortent en dehors du tunnel VPN
  • WebRTC leaks — le navigateur peut révéler ta vraie IP même avec VPN
# Tester les DNS leaks
$ curl https://dnsleaktest.com/api/    ← IPs révélées ?
# Split tunneling — router seulement certains trafics dans le VPN
# WireGuard : AllowedIPs = 10.0.0.0/8 → seul le réseau interne passe par le VPN
# AllowedIPs = 0.0.0.0/0 → tout le trafic passe par le VPN (full tunnel)

De l'Alerte à la Réponse — IDS vers SIEM

Suricata génère des événements, le SIEM les corrèle — construire un SOC minimal

🔔 Règles Suricata — exemples pratiques
# Détecter un scan Nmap SYN
alert tcp any any -> $HOME_NET any
(msg:"NMAP SYN Scan"; flags:S,12;
 threshold:type threshold,track by_src,count 20,seconds 1;
 sid:1000001; rev:1;)

# Détecter HTTP avec User-Agent Nmap
alert http any any -> any any
(msg:"Nmap HTTP UA"; content:"Nmap Scripting Engine";
 http_header; sid:1000002; rev:1;)

# Détecter téléchargement .exe depuis HTTP
alert http $EXTERNAL_NET any -> $HOME_NET any
(msg:"EXE download via HTTP";
 fileext:"exe"; sid:1000003; rev:1;)
📊 Stack SOC minimale
1
Suricata / Snort — NIDS, génère des alertes JSON dans /var/log/suricata/eve.json
2
Filebeat / Logstash — collecte et envoie les logs Suricata vers Elasticsearch
3
Elasticsearch — stockage et indexation des événements
4
Kibana — dashboards et visualisations, créer des alertes sur les signatures détectées
5
Wazuh — HIDS + SIEM all-in-one, excellent rapport qualité/prix pour PME africaines

Checklist Sécurité Réseau — Top 20 Priorités

Si tu ne peux faire que 20 choses pour sécuriser un réseau — ce sont celles-ci

🔴 Critique — À faire immédiatement
Changer tous les mots de passe par défaut des équipements
Mettre à jour le firmware de tous les équipements réseau
Firewall avec politique Deny-All par défaut
Segmenter le réseau avec des VLANs (minimum : users/servers/IoT/guest)
Fermer tous les ports non utilisés sur le firewall et les switches
Désactiver Telnet, FTP, HTTP admin — utiliser SSH et HTTPS
Activer DHCP Snooping et DAI sur les switches
HSTS + HTTPS forcé sur toutes les applications web
MFA sur tous les accès administratifs (VPN, jump server, cloud)
Sauvegardes chiffrées testées régulièrement (3-2-1)
🟡 Important — À faire dans les 30 jours
IDS/IPS déployé (Suricata + règles ET/Open) + alertes email
fail2ban ou CrowdSec sur tous les serveurs exposés
DNS over HTTPS ou DoT configuré sur les résolveurs
VPN pour tout accès distant (WireGuard recommandé)
WiFi : WPA3 ou WPA2-Enterprise · guest VLAN isolé
Logging centralisé — tous les équipements → syslog/SIEM
Scan Nmap régulier de sa propre infrastructure (mensuel)
SPF + DKIM + DMARC sur tous les domaines email
NTP synchronisé sur tous les équipements
Plan de réponse à incident documenté et testé

Certifications Réseau — Le Parcours Recommandé

Du débutant à l'expert — les certifications qui valorisent ton profil en Afrique et à l'international

CompTIA Network+
La certification réseau fondamentale, vendor-neutral. Couvre OSI, TCP/IP, sécurité réseau, troubleshooting. Reconnue mondialement, idéale pour débuter.
~360$ · 90 questions · 90 min · renouvellement 3 ans
CompTIA Security+
La certification sécurité de référence — complémentaire à Network+. Gouvernements, banques, grandes entreprises l'exigent souvent. Couvre tout le spectre de la sécurité.
~404$ · Prérequis recommandé : Network+ · Examen SY0-701
Cisco CCNA
La certification réseau Cisco — très demandée par les opérateurs africains (Orange, MTN, Airtel). Couvre le routage, le switching, VLANs, STP, OSPF, sécurité de base.
~330$ · Examen 200-301 · 120 questions · 120 min
Cisco CCNP Enterprise
Niveau avancé — routage avancé (OSPF, BGP, EIGRP), SD-WAN, infrastructure de campus, sécurité réseau avancée. Prérequis : CCNA. Ouvre les portes des grandes entreprises.
~400-500$ par examen · 2 examens · 3-5 ans d'expérience
Cisco CyberOps Associate
Spécialisation SOC — analyse de trafic, Wireshark, IDS/IPS, réponse aux incidents, SIEM. Parfait pour rejoindre un SOC après ce parcours NET.
~330$ · Très pratique · Idéal pour Blue Team · CV SOC Analyst
Mikrotik MTCNA
Certification Mikrotik — très populaire en Afrique où Mikrotik domine les ISP et PME. Routage, firewall, VPN, QoS, hotspot. Formation souvent disponible en Afrique.
~50$ · Examen en ligne · Très valorisé par les FAI africains
