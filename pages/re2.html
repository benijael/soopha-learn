Pourquoi Apprendre l'Assembleur en 2026 ?

Le compilateur ne ment pas — l'assembleur montre ce qui se passe vraiment

🎯 Sans asm, tu es aveugle
  • Ghidra produit un pseudo-code approximatif — l'asm est la vérité absolue
  • Les optimisations du compilateur (-O2, -O3) transforment radicalement le code C — seul l'asm dit ce qui s'exécute vraiment
  • Les vulnérabilités bas niveau (buffer overflow, format string, UAF) ne se voient qu'en asm
  • En CTF, la logique de validation du flag est toujours dans l'asm — le décompilateur peut se tromper
✅ Ce que tu peux faire après ce module
  • Lire et comprendre n'importe quelle fonction désassemblée
  • Identifier immédiatement les boucles, conditions et appels de fonction
  • Suivre le flux de données entre les registres et la mémoire
  • Poser un breakpoint GDB au bon endroit grâce à l'adresse asm
  • Comprendre pourquoi le décompilateur Ghidra affiche parfois du code étrange
"L'assembleur, c'est la langue maternelle du CPU. Tout ce qui s'exécute sur ta machine parle assembleur. En l'apprenant, tu parles directement au processeur — sans intermédiaire, sans mensonge."

// Slide 04 · RSP · RBP · RIP · RFLAGS

Registres Spéciaux — Le Chef d'Orchestre du CPU

Ces registres contrôlent le flux d'exécution — les plus importants en RE

RSP — Stack Pointer
Pointe toujours vers le sommet de la pile (stack). Décrémenté par PUSH, incrémenté par POP. RSP - offset = accès aux variables locales. Ne jamais modifier directement sans raison.
x/8xg $rsp → voir le sommet de la pile dans GDB
RBP — Base Pointer
Base du stack frame courant. Les variables locales sont à RBP-offset (ex: [rbp-0x8]) et les arguments anciens à RBP+offset. Sert de référence stable dans la fonction.
[rbp-0x8] = variable locale 1 · [rbp-0x10] = variable locale 2
RIP — Instruction Pointer
Adresse de la prochaine instruction à exécuter. Ne peut pas être modifié directement en x86-64 (contrairement à x86-32). Modifié par JMP, CALL, RET, conditions.
p $rip → adresse actuelle · set $rip = 0x1234 dans GDB
RFLAGS — Registre de drapeaux
Bits qui enregistrent le résultat d'une comparaison. ZF (Zero Flag) = résultat nul. CF (Carry) = débordement non signé. SF (Sign) = résultat négatif. OF (Overflow) = débordement signé.
CMP rax, 0 → si rax=0 : ZF=1 → JE saute
XMM0–XMM15 — Registres SIMD
Registres 128-bit pour les opérations SIMD (Single Instruction Multiple Data). Utilisés par les fonctions mathématiques, les chaînes optimisées, et parfois le chiffrement.
memcpy et strcmp optimisés utilisent souvent XMM0
Taille des registres
Un même registre physique selon la taille utilisée : RAX (64-bit) → EAX (32-bit, efface les 32 bits hauts) → AX (16-bit) → AL/AH (8-bit). Crucial pour comprendre les mouvements de données.
mov eax, 1 → RAX = 0x0000000000000001
// Slide 05 · Instructions clés · MOV LEA ADD XOR · QCM requis

Instructions Fondamentales — Le Vocabulaire de l'Assembleur

Les 10 instructions que tu verras dans CHAQUE fonction désassemblée

MOV dst, src
Déplace une valeur d'une source vers une destination. Registre → registre, mémoire → registre, immédiat → registre. L'instruction la plus fréquente de l'assembleur.
mov rax, rdi   ; rax = rdi
mov [rbp-8], rax ; écrire en mémoire
LEA dst, [expr]
Load Effective Address — charge l'ADRESSE calculée, pas la valeur à cette adresse. Utilisé pour des calculs d'adresse et souvent par le compilateur pour des multiplications rapides.
lea rdi, [rbp-0x40] ; adresse du buffer local
lea rax, [rax+rax*2] ; rax = rax * 3
ADD / SUB
Addition et soustraction. Modifient les flags RFLAGS (ZF, CF, OF, SF). sub rsp, 0x40 = réserver 64 octets pour les variables locales (très courant dans les prologues).
add rax, 1   ; rax++
sub rsp, 0x40 ; allouer 64 octets sur la stack
XOR dst, src
OU exclusif bit à bit. Deux usages : (1) zeroing rapide : xor eax, eax = mettre RAX à 0. (2) Encodage/décodage XOR en CTF. Le compilateur utilise souvent xor eax, eax plutôt que mov eax, 0.
xor eax, eax       ; rax = 0 (zeroing)
xor al, 0x42       ; décodage XOR
IMUL / MUL / DIV
Multiplication et division. IMUL = signé. MUL = non signé. La division est rarement générée directement — le compilateur utilise des shifts et multiplications magiques pour les constantes.
imul rax, rdx, 0x1d ; rax = rdx * 29
AND / OR / NOT / SHL / SHR
Opérations bit à bit et shifts. SHL = shift left (multiplier par 2^n). SHR = shift right (diviser par 2^n). Très utilisés dans les algorithmes de chiffrement et les flags de bits.
shl rax, 3 ; rax = rax * 8
and rax, 0xff ; garder seulement le byte bas
// Slide 06 · Stack · PUSH · POP · Fonctionnement

La Pile (Stack) — La Mémoire des Fonctions

Comprendre la stack = comprendre comment toute fonction gère son espace

📚 Fonctionnement LIFO
  • La stack grandit vers le bas — les adresses diminuent quand on empile
  • PUSH rax → RSP -= 8, puis [RSP] = RAX (écriture en tête de pile)
  • POP rax → RAX = [RSP], puis RSP += 8 (lecture et libération)
  • Chaque appel de fonction crée un nouveau stack frame — espace isolé pour variables locales et état des registres
  • Si RSP sort du segment stack → Stack Overflow (crash ou exploitation)
🏗️ Structure d'un stack frame
; Adresses décroissantes vers le bas
Haute adresse
┌──────────────────┐
│ Arguments caller │ ← [rbp+0x10]
│ Adresse retour │ ← [rbp+0x08] (CALL)
│ Ancien RBP sauvé │ ← [rbp] (PUSH rbp)
│ Variable locale 1 │ ← [rbp-0x08]
│ Variable locale 2 │ ← [rbp-0x10]
│ Buffer (tableau) │ ← [rbp-0x50]
│ ←── RSP (sommet) │
└──────────────────┘
Basse adresse

CALL & RET — Mécanique des Appels de Fonctions

Chaque fonction suit le même schéma — le reconnaître instantanément

📞 Ce que CALL fait réellement
  • 1. PUSH l'adresse de l'instruction suivante sur la stack (adresse de retour)
  • 2. JMP vers l'adresse de la fonction appelée (modifie RIP)
  • Résultat : RSP -= 8 et RIP = adresse de la fonction
  • C'est pourquoi les buffer overflows sur la stack peuvent écraser l'adresse de retour et hijacker le flux d'exécution
🔄 Ce que RET fait réellement
  • 1. POP l'adresse de retour depuis le sommet de la stack
  • 2. JMP vers cette adresse (retour à l'appelant)
  • Résultat : RSP += 8 et RIP = adresse de retour
  • Si la stack a été corrompue (overflow), RET saute vers une adresse contrôlée par l'attaquant
; Prologue standard — IDENTIQUE dans presque toutes les fonctions
push rbp               ; sauvegarder le frame pointer de l'appelant
mov  rbp, rsp           ; créer le nouveau stack frame
sub  rsp, 0x40          ; réserver 64 octets pour les variables locales

; ... corps de la fonction ...

; Épilogue standard
leave                  ; = mov rsp, rbp + pop rbp
ret                    ; pop rip (retour à l'appelant)
// Slide 08 · CMP · TEST · JMP · JE · JNE · QCM requis

Contrôle de Flux — CMP, TEST et les Sauts Conditionnels

Comment le processeur prend des décisions — la clé pour identifier les conditions if/else

⚖️ CMP et TEST — Comparer sans modifier
  • CMP a, b — effectue (a - b) et positionne les flags SANS stocker le résultat. ZF=1 si a==b, CF=1 si a<b (non signé)
  • TEST a, b — effectue (a AND b) et positionne les flags. TEST rax, rax → ZF=1 si rax==0. Idiome très fréquent pour tester la nullité
  • CMP = pour comparer deux valeurs · TEST = pour tester des bits ou vérifier le zéro
🔀 Sauts conditionnels essentiels
JE / JZ  ; Jump if Equal / Zero Flag = 1
JNE / JNZ  ; Jump if Not Equal / ZF = 0
JG / JNLE ; Jump if Greater (signé)
JGE / JNL ; Jump if Greater or Equal
JL / JNGE ; Jump if Less (signé)
JLE / JNG ; Jump if Less or Equal
JA / JB  ; Above / Below (non signé)
JMP ; Saut inconditionnel
; Pattern classique : if (rax == 0) goto .success
cmp rax, 0                    ; rax - 0 → flags
je .success                    ; si ZF=1 → .success

; Pattern : while (i < len) → la boucle CTF classique
cmp rcx, rdx                   ; i vs len
jge .loop_end                   ; si i >= len → sortir

System V AMD64 ABI — La Convention d'Appel Linux

Comment les arguments sont passés et récupérés — essentiel pour tracer les appels de fonction

📦 Ordre des arguments (Linux/macOS)
// Fonction C : int check(char *s, int len, int key)
// En assembleur avant le CALL :
RDI = s          ; 1er argument — pointeur string
RSI = len        ; 2e argument — longueur
RDX = key        ; 3e argument — clé XOR
RCX = 4e arg
R8 = 5e arg
R9 = 6e arg
7e+ args → sur la stack (au-delà de 6)
RAX = valeur de retour de la fonction
📋 Registres préservés (Callee-saved)
  • Caller-saved (tu peux les utiliser librement) : RAX, RCX, RDX, RSI, RDI, R8–R11
  • Callee-saved (la fonction appelée DOIT les restaurer) : RBX, RBP, R12–R15
  • En RE : si tu vois PUSH RBX au début d'une fonction → RBX va être utilisé intensivement
  • Les compilateurs optimisés ignorent parfois le frame pointer (RBP) pour gagner un registre
"Mémo infaillible : RDI RSI RDX RCX R8 R9. Sur Linux, les 6 premiers arguments d'un appel strcmp(s1, s2) → RDI=s1, RSI=s2. Mets un breakpoint avant CALL strcmp, affiche x/s $rdi et x/s $rsi — tu lis les deux chaînes comparées."

Lire une Boucle en Assembleur

Le pattern for/while en asm — le plus fréquent dans les challenges RE

📖 Exemple complet : boucle XOR (challenge 2 SooPHA CTF)
; int check_flag(char *input) {
;   for (int i = 0; i < 29; i++) {
;     if ((input[i] ^ 0x42) != encoded[i]) return 0;
;   } return 1; }

xor eax, eax                   ; i = 0
.loop:
cmp eax, 0x1d                  ; i < 29 (0x1d)
jge .done                       ; si i >= 29 → exit
movzx ecx, byte [rdi+rax]       ; ecx = input[i]
xor ecx, 0x42                  ; ecx ^= 0x42
movzx edx, byte [rip+encoded+rax] ; edx = encoded[i]
cmp ecx, edx                   ; input[i]^0x42 vs encoded[i]
jne .fail                       ; si différent → fail
inc eax                         ; i++
jmp .loop                       ; retour au début
.done: mov eax, 1 · ret          ; return 1 (succès)
.fail: xor eax, eax · ret        ; return 0 (échec)

Traduire Mentalement l'Assembleur en Logique Métier

Le but n'est pas de réciter les mnémotechniques, mais de reconstruire l'intention du développeur

cmp rdi, rsi / je ok
Comparer deux valeurs puis sauter si elles sont égales.
if (a == b) goto ok;
test rax, rax / jne next
Vérifier si une valeur est nulle sans la modifier.
if (ptr != NULL) goto next;
xor eax, eax
Idiomatique : remettre un registre à zéro rapidement.
return 0; ou int i = 0;
lea rax, [rbp-0x20]
Calculer une adresse, souvent pour un buffer local.
char *buf = &stack_buf[0];
🧠 Question utile
  • Qu'est-ce qui est comparé ?
  • Qu'est-ce qui est initialisé ?
  • Quel registre transporte un argument ou un retour ?
  • Quelle structure haut niveau ce pattern évoque ?
🎯 Réflexe
  • Ne lis pas l'asm ligne par ligne sans contexte.
  • Lis des blocs logiques : préparation, comparaison, branche, retour.
  • Pense toujours : “si je devais le réécrire en C, à quoi ça ressemblerait ?”

mov rbp, rsp

mov [rbp-0x8], rdi            ; local_1 = arg1 (pointeur)

// Slide 13 · Mémo · Erreurs fréquentes · Synthèse

Mémo Opérationnel x86-64

Ce que tu dois reconnaître en quelques secondes avant d'ouvrir Ghidra

📌 Patterns instantanés
  • push rbp / mov rbp, rsp → entrée de fonction classique
  • cmp / je / jne → condition et embranchement
  • call strcmp / strlen / puts → logique de validation ou d'affichage
  • movzx → lecture d'octet non signé, très fréquent dans les boucles de flag
⚠️ Erreurs fréquentes
  • Confondre adresse et valeur.
  • Oublier la calling convention et attribuer le mauvais rôle à un registre.
  • Lire une boucle sans repérer d'abord le compteur, la condition et l'incrément.
